# TrainTimer

本地命令行魔方计时器，支持成绩记录、毫秒级显示、15 秒观察倒计时，以及项目内置的 TNoodle CLI 打乱公式生成。

## 运行

网页界面：

```bash
npm run web
```

默认会启动在 `http://127.0.0.1:3211`，如果端口被占用会自动顺延；也可以直接打开终端输出的地址。网页需要本地服务提供 API，不建议直接用 `file://` 打开 `public/index.html`。

命令行界面：

```bash
npm start
```

开启观察时间：

```bash
npm start -- --inspection
```

关闭观察时间：

```bash
npm start -- --no-inspection
```

查看最近成绩：

```bash
npm start -- history
```

## 按键

- 准备状态：开启观察时按 `Space` 开始观察；未开启观察时持续按住 `Space` 超过 0.5 秒后松开才开始计时，短按不会启动。
- 观察状态：持续按住 `Space` 超过 0.5 秒后松开才开始计时，短按不会启动；第 8 秒和第 12 秒会响铃提醒；超过 15 秒开始计时会自动记 `+2`，超过 17 秒会自动记 `DNF`。
- 计时状态：按任意键结束并记录成绩。
- 已记录状态：按 `O` 把上一把设为 OK，按 `2` 设为 `+2`，按 `D` 设为 `DNF`，按 `Backspace/Delete` 删除上一把。
- 退出：按 `q`、`Esc` 或 `Ctrl+C`。

## 网页功能

- 计时器、观察倒计时和成绩统计；计时偏好支持计时中隐藏数字、起步冻结 0.2s/0.5s/1s/2s，以及删除成绩前确认开关，适合专注训练、减少开表瞬间干扰并按个人习惯管理误删确认；网页快捷键支持下一把、新打乱、锁定打乱、观察开关、统计详情、全部成绩和偏好弹窗，偏好页内提供快捷键速查。
- 支持会话新建、复制、合并、重命名和删除，并按当前会话显示次数、最佳、平均、mo3、ao5、ao12、最佳 mo3、最佳 ao5、最佳 ao12、统计详情、当前筛选列表统计、选中成绩统计、会话概览与快速切换、最佳单次/滚动平均定位、ao50/ao100 等更多滚动平均、最近成绩、成绩表逐条 ao5/ao12 和 PB/ao50/ao100 刷新标记、mo3/ao5/ao12/ao50/ao100 明细查看与复制、单次/mo3/ao5/ao12/ao50/ao100/TPS 趋势图、有效成绩分布图和完整统计摘要复制。
- 历史成绩读取同一个本地 JSON 文件。
- 打乱公式由内置 TNoodle CLI 生成，网页可直接切换 2x2、3x3、4x4、5x5、6x6、7x7、Clock、Skewb 和 Square-1 打乱类型；每个会话会记住自己的打乱类型，成绩也会保存对应打乱类型；支持锁定当前打乱，下一把继续使用同一条公式做专项训练。
- 打乱结果预览优先使用 TNoodle `draw` 输出的 2D 展开图，确保预览状态和 TNoodle 打乱定义一致；TNoodle 预览不可用时，3x3 会回退到本地状态渲染，其他类型会提示需要 TNoodle 预览。
- 算法训练支持 PLL、2-Look OLL 和本地自定义公式集练习，按全部/复习/未练/薄弱筛选，并可按名称、分组、公式和提示本地搜索；复习队列会自动收录未练、准确率低、超过 7 天未练或计时偏慢的公式；训练记录掌握率、连续掌握次数，并可对当前公式做独立计时练习，保存最近、最佳和平均用时。
- 成绩管理支持手动录入并指定复原时间、上一把快速 OK/+2/DNF/删除、单条删除、详情删除、选中删除、单条/批量标记 OK/+2/DNF、批量打乱类型、批量标签、批量备注、清空当前会话、删除会话后撤销恢复、撤销最近一次删除/移动/合并/标记/类型/标签/备注/详情编辑/导入、查看当前会话或全部会话成绩，列表会直接显示打乱类型、蓝牙来源、标签和备注摘要；按成绩/罚时/标签/备注/打乱类型/打乱/会话筛选成绩、按日期范围筛选成绩、用今天/本周/本月快速日期筛选成绩、按单次 PB、mo3/ao5/ao12/ao50/ao100 刷新纪录筛选成绩、按专用打乱类型/罚时/手动或蓝牙来源/标签或无标签筛选成绩、一键清除全部筛选、按日期/成绩/罚时/打乱类型/来源/会话/标签/备注排序、当前筛选列表一键复制为文本表格、选中成绩移动到其他会话、单次成绩详情、同会话上一条/下一条详情导航、原始成绩编辑、复原时间编辑、详情罚时编辑、打乱类型编辑、打乱公式编辑、备注、标签、打乱公式复制、单次成绩详情复制、当前会话/当前筛选列表/选中成绩/全部数据的 JSON/CSV 导出、当前会话/当前筛选列表/选中成绩/全部数据的 csTimer CSV 和带会话打乱类型的 csTimer JSON 备份导出，以及带预览确认的 TrainTimer JSON/CSV、csTimer CSV 和常见 csTimer JSON 备份导入；TrainTimer CSV 和 csTimer JSON 导入会保留打乱类型，TrainTimer CSV 导入会保留蓝牙转动数和 TPS 统计；导入或合并时会自动处理重复成绩 ID，避免后续编辑或删除误命中多条。
- 蓝牙魔方连接使用浏览器 Web Bluetooth；需要支持 Web Bluetooth 的浏览器和真实设备。网页启动时会预检 Web Bluetooth、设备选择器和安全上下文，不可用时会直接禁用连接入口并在状态栏说明原因。网页会枚举 GATT 服务、订阅 notify/indicate 特征，并在状态栏、蓝牙日志和控制台显示设备发出的原始数据。默认连接会按常见智能魔方名称筛选设备，“兼容扫描”会列出更多蓝牙设备，方便连接名称不在默认列表里的魔方；浏览器保存过授权后，可用“重连”直接连接已授权设备，也可以用独立“断开”按钮结束当前连接。若设备提供标准 Battery Service，会读取并显示电量；GoCube / Rubik's Connected 会通过 Nordic UART 自动请求状态和电量，并解析其二进制转动包；Giiker / Mi Smart 会解析 20 字节状态包中的最新转动，并向电量服务发送查询，重复的状态包会被识别为重复数据，避免同一个最新转动被计入多次。日志也会尝试把通知中的标准转动记号、JSON 转动字段或分隔符转动串解析为转动流，并支持复制或导出 JSON 诊断日志；正式计时开始前的预备转动只写入日志，不计入本把成绩，计时中解析到的转动会应用到当前打乱状态上显示是否已复原，蓝牙日志会同步显示当前魔方展开状态，检测到已复原会自动停表并保存成绩。成绩详情、复制摘要和导出数据会保留手动/蓝牙停表来源、本把蓝牙转动序列、转动数、TPS、蓝牙设备名、解析协议、通知来源和单次蓝牙复盘状态，便于继续接入更多具体品牌协议和排查设备数据。

## 成绩文件

默认写入：

```text
~/.train-timer/solves.json
```

可以用环境变量改位置：

```bash
TRAIN_TIMER_HISTORY=/path/to/solves.json npm start
```

## TNoodle 打乱

TNoodle 是 WCA 官方打乱程序项目：https://github.com/thewca/tnoodle

项目内置了 `vendor/tnoodle-cli-1.1.1.jar`，默认通过它调用 TNoodle 核心打乱引擎生成打乱公式。需要 Java 11 或更高版本。

也可以用外部 TNoodle 兼容 CLI 覆盖默认来源：

```bash
TNOODLE_CMD=/path/to/tnoodle npm start
```

或者调用可执行 jar：

```bash
TNOODLE_JAR=/path/to/tnoodle-cli.jar npm start
```

命令行可通过环境变量指定 puzzle，默认是 `three`：

```bash
TNOODLE_PUZZLE=three npm start
```

如果本机未安装 TNoodle 命令或未配置 `TNOODLE_JAR`，程序会使用本地备用 3x3 随机步打乱，并在界面中明确提示。正式 WCA 用途应使用当前官方 TNoodle 程序。
如果需要强制禁用项目内置 JAR 来测试备用路径：

```bash
TRAIN_TIMER_DISABLE_BUNDLED_TNOODLE=1 npm start
```
