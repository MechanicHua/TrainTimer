# TrainTimer Logo Exploration · Round 1

## 目标与判断

TrainTimer 是面向速拧训练的本地计时与数据工具。这次重做不是对旧 Logo 的修饰，而是重新建立一个能够长期使用的品牌识别系统。用户希望品牌第一眼呈现「专业竞速工具」，并明确允许彻底放弃旧标志中的魔方、秒表、环形轨迹、彩色贴纸等元素。新标志不需要逐字解释产品功能，但必须同时让人感到精确、快速、可靠，像一件经过校准的训练仪器，而不是休闲玩具、普通电子表或电竞直播素材。

## 使用场景与观看距离

首要场景是 macOS App Icon：Dock、Finder、Spotlight、启动器窗口内约 48 px 的应用标记，以及 16–32 px 的小尺寸系统显示。其次是 Chrome favicon、网页品牌标记、About 面板和横版 TrainTimer 字标。设计应先通过 32 px 轮廓测试，再考虑 1024 px 的材质细节；不能依赖细刻度、长文字、九宫格小块、复杂透视或多圈速度线来成立。

## macOS 26 共同设计系统

- 画布以 1024×1024 为共同尺寸；概念预览可展示系统圆角遮罩，但最终分层源稿应保留未预先遮罩的方形图层。
- 构图以正面、稳定、光学居中为主。核心符号占画面约 65–72%，四周留出足够呼吸区，避免靠边细节被系统遮罩或缩放损坏。
- 每案最多三个清晰平面：背景底板、主符号、一个局部信号层。材质可以有轻微半透明、折射、内高光和短阴影，但不能把 Liquid Glass 当作装饰铺满。
- 形状采用粗线、圆角、明确负形。避免锐利针状角、极细线、写实 3D 物体、完整 UI 截图和复杂文字。
- 色彩以石墨、钛灰、烟黑等中性色建立专业感，每案只允许一个主信号色。信号色应表达测量、分段或动作，不做彩虹、紫粉科技渐变或电竞霓虹。
- 小尺寸版允许主动删减高光、暗纹、信号点和次级层；必须保持与大图相同的核心轮廓，而不是机械等比缩放。
- 横版字标属于第二阶段。本轮仅用克制的临时排版展示图形与名称的关系，不把字体预览误当最终字标。

## 三条独立探索路线

### A · TT Gate / 计时门

从 TrainTimer 的双 T 出发，把字母理解为竞速计时门与分段闸门。符号应轴线稳定、宽重心、接近专业硬件品牌；负形或极少量信号点承载「split」含义。配色使用石墨底、钛银主体与单一竞速橙。最大风险是像交通、铁路或服装字母标，因此造型必须具有计时分割关系，避免普通并排 TT。

### B · Zero Point / 零点

从七段数显和归零时刻出发，构造一个高度抽象的 `0` 与分秒信号。它应让用户无需读名称就感到精确计时，同时避免直接画 `0.00` 或复制计算器界面。配色使用深海军蓝或烟熏玻璃底、冰青信号。最大风险是像计算器、电子表或泛用计时器，因此需要独特的段形比例与清晰的「零点」切口。

### C · Split Turn / 分层转位

从速拧的核心动作「转层」出发，但不画完整魔方。使用少量厚实面片、错位或中心负形冻结一次高速转位，让图形同时拥有结构、动作和测量感。配色使用深钛底、冷白主体与单一钴蓝信号。最大风险是像同步、快门或拼图图标，因此应采用块状转位而非环形箭头，并在 32 px 时保持独有轮廓。

## 第一轮交付与决策方式

每条路线交付一个独立 SVG App Icon 概念，并在统一比较页中展示：大尺寸主视觉、浅色桌面、深色桌面、64 px、32 px、单色轮廓和临时横版组合。第一轮不替换 `public/`、App Bundle 或 `.icns`。用户看到真实视觉后，可以选择一案深化，也可以混合「A 的构图 + B 的色彩」等具体特征。确定方向后再制作主标志、完整字标、favicon、默认/深色/单色版本、完整 iconset 与可复现的打包流程。
